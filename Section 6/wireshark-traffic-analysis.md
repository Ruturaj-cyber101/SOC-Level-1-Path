# Wireshark: Traffic Analysis

## Overview
This room focuses on analyzing captured network traffic using Wireshark. It teaches how security analysts examine packet data to identify suspicious activity and investigate potential network-based attacks.

## Key Concepts
- Traffic analysis involves examining captured packets to understand communication between devices.
- Analysts look for unusual patterns, suspicious IP addresses, and abnormal protocol behavior.
- Packet inspection helps identify malicious activity within network traffic.

## Important Techniques
- Inspecting packet details
- Following TCP streams
- Analyzing DNS queries and responses
- Identifying suspicious connections
- Reviewing communication between hosts

## Common Indicators of Suspicious Traffic
- Communication with unknown or malicious IP addresses
- Unusual DNS requests
- Large data transfers to external systems
- Unexpected protocols or ports being used

## SOC Analyst Perspective
SOC analysts perform traffic analysis to investigate alerts, detect malware communication, and identify possible data exfiltration or command-and-control (C2) traffic within a network.

## Key Takeaways
- Traffic analysis helps uncover hidden threats in network communication.
- Packet-level visibility allows analysts to understand attacker behavior.
- Wireshark is an essential tool for network-based investigations.
